## React 結構

首頁維持現有頁面切換方式，不引入路由器或後端。預計拆分為：

- `Home`
  - 負責首頁整體版面、背景與導航事件。
- `RainbowHomeTitle`
  - 顯示標題圖片並提供可存取的標題文字。
- `HomeLearningGrid`
  - 依資料陣列渲染五張卡片。
- `HomeLearningCard`
  - 封裝卡片圖片、操作狀態、即將開放標籤與鍵盤焦點樣式。
- `HomeMascots`
  - 負責熊兔位置與裝飾動畫。

卡片資料繼續 hardcode 在元件或資料常數中，不建立資料庫或正式 JSON：

```js
const homeLearningItems = [
  { id: 'zhuyin', label: '注音學習', image: cardZhuyin, enabled: true },
  { id: 'number', label: '數字學習', image: cardNumber, enabled: true },
  { id: 'english', label: '英文學習', image: cardEnglish, enabled: false },
  { id: 'shape', label: '形狀配對', image: cardShape, enabled: false },
  { id: 'logic', label: '邏輯思考', image: cardLogic, enabled: false },
];
```

## 操作行為

- 「注音學習」進入既有注音練習頁。
- 「數字學習」進入既有數字練習頁。
- 「英文學習」、「形狀配對」、「邏輯思考」顯示「即將開放」，不切換到不存在的頁面。
- 聲音按鈕沿用現有開關狀態與操作。
- 家長模式按鈕沿用現有家長頁入口。
- 可操作卡片提供 hover、focus 與 active 狀態：
  - hover：向上移動約 6px，放大約 1.02 至 1.04。
  - active：縮小至約 0.98，模擬柔軟按壓。
  - 鍵盤 focus：清楚但不刺眼的白色與藍色雙層焦點環。
- 首次顯示可使用輕微交錯進場，不使用長時間循環晃動。
- `prefers-reduced-motion` 開啟時停用位移與進場動畫。

## 響應式排列

### 桌機與 iPad 橫向

- 主容器最大寬度 1440px，水平置中。
- 使用六欄網格完成 3＋2 排列：
  - 第一排三張卡片各占兩欄。
  - 第二排兩張卡片各占兩欄並置中。
- 熊兔靠近卡片群組兩側，不貼齊視窗邊緣。
- 版面高度需納入 `100dvh`、安全區與加入主畫面後的底部 Home Indicator 空間。
- 不允許底部卡片或操作區被視窗裁切。

### 小尺寸橫向畫面

- 卡片區改為水平滑動選單。
- 使用 CSS scroll snap，讓每次滑動停在完整卡片位置。
- 顯示左右切換按鈕，並支援手指滑動。
- 熊兔縮小或移至背景層，避免遮住卡片。

### 直式畫面

- 不顯示強制旋轉遮罩，也不阻止使用者操作。
- Web App manifest 將預設方向設為 `landscape`。
- 主要視覺與尺寸只針對橫向最佳化；直式時使用安全的單欄或水平卡片滑動排版，避免內容溢出。

## 可存取性

- 五張卡片使用真正的 `<button>`，不是只有圖片熱區。
- 卡片圖片提供對應替代文字或由按鈕的 `aria-label` 說明功能。
- 未開放卡片使用 `aria-disabled="true"`，同時保留可辨識的「即將開放」文字。
- 聲音與家長模式觸控範圍至少 56px。
- 不將關鍵功能文字只留在背景圖片中；標題與卡片名稱在 DOM 中保留可存取文字。

## 驗證方式

1. 執行現有自動化測試與 Vite production build。
2. 桌機橫向檢查 1440×900 與 1920×1080。
3. iPad Air 4 橫向檢查 1180×820，並考慮加入主畫面後的安全區。
4. 小尺寸橫向檢查 844×390 或相近尺寸，確認卡片可滑動且不被裁切。
5. 檢查注音、數字與家長模式入口。
6. 檢查三張未開放卡片不會進入空白頁。
7. 檢查聲音按鈕、鍵盤 focus、觸控按壓與 `prefers-reduced-motion`。
8. 檢查瀏覽器主控台無錯誤與遺失素材請求。

## 驗收條件

- 首頁視覺明確符合使用者提供的新版圖片，不再呈現低多邊形風格。
- 首頁不是整張靜態背景圖，所有主要按鈕均可獨立操作。
- 桌機與 iPad 橫向正確顯示 3＋2 卡片排列。
- 小尺寸橫向畫面使用可操作的水平滑動卡片。
- 注音、數字、家長模式與聲音功能維持正常。
- 三個未完成項目清楚顯示「即將開放」。
- 首頁在目標橫向尺寸沒有按鈕被裁切、重疊或超出安全區。
